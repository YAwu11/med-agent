# ADR-036: 脑肿瘤 MRI 4 步级联流水线（全 API 模式）

## 问题背景

现有 `brain_mcp.py` 分析器仅支持 2D 切片的 YOLO-Seg 实例分割，无法处理 NIfTI 3D 体积数据，缺乏解剖学定位能力。
本方案实现一套完整的 **3D 分割 → 空间定位 → 2D 渲染 → 受控报告** 级联流水线。

### 核心约束

- **所有大模型调用走硅基流动 API**（Qwen2.5-VL-72B 或 Qwen3-VL-32B），不依赖本地 GPU 跑 LLM
- **本地 GPU 仅用于 nnU-Net 推理**（3D 医学分割，约 4-6GB VRAM）
- **脑图谱配准使用 ANTsPyx 仿射变换**（纯 CPU），不用 SynthSeg

---

## 架构决策

### ADR-036-1: 全 API 模式 vs 本地 VLM

| 维度 | 本地 VLM (Qwen2-VL-7B) | 硅基流动 API (Qwen2.5-VL-72B) |
|:---|:---|:---|
| 质量 | 7B 级别，医学文本生成能力有限 | 72B 级别，报告质量显著更好 |
| 延迟 | ~5s（本地推理） | ~3-8s（网络 + 推理） |
| VRAM | 需要 ~16GB（与 nnU-Net 争抢） | 0 |
| 运维 | 需要维护模型版本 | 平台托管 |

**决策：全部走 API。** 理由：72B 模型报告质量远优于本地 7B，且释放 GPU 给 nnU-Net 独占。

### ADR-036-2: MNI 仿射配准 vs SynthSeg

| 维度 | SynthSeg | MNI + AAL 图谱 |
|:---|:---|:---|
| 大肿瘤鲁棒性 | ❌ 会被肿瘤变形误导 | ✅ 仿射变换不受局部变形影响 |
| 依赖 | FreeSurfer（重量级） | ANTsPyx（pip install） |
| 速度 | ~30s | ~15s |

**决策：MNI 仿射配准 + AAL 图谱。** 用 `antspyx` 纯 Python 库，仅做仿射（12 DOF），不做非线性。

## 系统架构

```text
                      ┌──── 硅基流动 API ────┐
                      │  Qwen2.5-VL-72B     │
                      │  (Step 4 报告生成)   │
                      └─────────▲───────────┘
                                │ HTTPS
                                │
┌───────── 1_core_orchestrator ──────────────────────────────────┐
│                                                                │
│  brain_mcp.py (调度入口)                                       │
│       │                                                        │
│       ▼                                                        │
│  brain_nifti_pipeline.py  ← 新增核心模块                       │
│       │                                                        │
│       ├─ Step 1: nnU-Net 推理 (本地 GPU, ~6GB)                 │
│       │     └─ 调用 nnunetv2 predict API                       │
│       │                                                        │
│       ├─ Step 2: 空间计算 (纯 CPU)                             │
│       │     ├─ 2a: 体积精算 (numpy × spacing)                  │
│       │     ├─ 2b: ANTsPyx 仿射配准 → MNI → AAL 图谱查询       │
│       │     └─ 2c: 临床空间关系计算 (NEW)                      │
│       │           ├─ 跨中线判定 (MNI x=0 矢状面)               │
│       │           ├─ 脑干最近距离 (mm)                         │
│       │           └─ 脑室受压比 (同侧 vs 对侧体积比)           │
│       │                                                        │
│       ├─ Step 3: 2D 切片渲染 (纯 CPU)                          │
│       │     ├─ 最大截面选择                                    │
│       │     ├─ T1ce + FLAIR 对比图拼接                         │
│       │     └─ R/L 标注 + 轮廓叠加 → PNG                       │
│       │                                                        │
│       └─ Step 4: 受控报告生成 (硅基流动 API)                   │
│             ├─ 系统强制前置数据（位置+体积+空间关系）          │
│             ├─ 2D 切片图片（Base64）                           │
│             └─ JSON Schema 结构化输出 + 事后对账               │
│                                                                │
└────────────────────────────────────────────────────────────────┘
```

## 后续开发组件清单

### 后端 — `backend/app/gateway/services/`

#### [NEW] `brain_nifti_pipeline.py` — 核心流水线

> [!IMPORTANT]
> **关键架构决策：Step 1-3 与 Step 4 拆分执行。**
> - **上传时自动执行 Step 1-3**：产出分割结果 + 空间数据 + 渲染切片，存入 `structured_data`
> - **医生审核确认后才执行 Step 4**：用医生确认/修正后的数据调用硅基流动 API 生成报告
>
> 这样确保报告使用的是医生审核过的数据，不是未经验证的机器输出。
